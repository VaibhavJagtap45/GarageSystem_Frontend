1 Project Overview

ApnoGarage is a full-featured mobile garage management system built for auto repair businesses. It connects three types of users — garage owners, repair members (mechanics), and customers — on a single platform, enabling end-to-end management of repair orders, inventory, invoices, and team operations.

Core Purpose: Replace paper-based job cards, spreadsheet invoicing, and manual parts tracking with a single mobile app that works for the garage owner, their team, and their customers simultaneously.

What the System Does

  • Create and manage repair orders from vehicle drop-off to completion
  • Assign jobs to mechanics and track their real-time status
  • Generate professional PDF invoices with garage logo
  • Manage spare parts inventory with stock alerts
  • Track all customer vehicles and service history
  • Handle vendor purchase orders and stock-in records
  • Generate financial reports — daily, monthly, income/expense, tally export
  • Send invoices via WhatsApp/SMS directly from the app
  • Allow customers to browse services, track their own orders, and view invoices
  • Allow mechanics to accept/reject assignments and update job status on the go
  • OTP-based login (no passwords to remember or forget)

Order Lifecycle

📋 Created Order raised by owner or customer
🔧 Assigned Mechanic accepts the job
⚙️ In Progress Work underway
🚗 Vehicle Ready Repair done, pending collection
✅ Completed Invoice raised & payment collected

4 System Architecture

ApnoGarage follows a standard Client–Server architecture. The mobile app (React Native) communicates with the Node.js REST API over HTTPS. The API reads/writes data from MongoDB and issues JWT tokens for authentication.

Authentication Flow

1

Enter Phone Number

User types their 10-digit mobile number on the Login screen.

2

Receive OTP via SMS

A 6-digit OTP is sent to the registered phone number. Valid for 5 minutes. Maximum 5 attempts allowed.

3

Verify OTP

User enters the OTP. Server validates and returns an access token + refresh token.

4

Role-Based Redirect

Based on the user's role (owner / member / customer), the app navigates to the correct portal automatically.

5 Owner Portal — User Guide

Who uses this: The garage owner or manager. Login with the owner's registered phone number.

5.1 First-Time Setup (New Owner)

1

Login with Phone Number + OTP

Enter your registered mobile number. You will receive an SMS OTP. Enter it to proceed.

2

Complete Garage Profile

First-time users are taken to the Garage Setup screen. Fill in: Garage Name, Owner Name, Address, City, State, Contact Number, Garage Type, and GST number (optional).

3

Upload Garage Logo

Go to More → Garage Profile → Edit. Tap "Choose from Gallery" to upload your logo. It appears on all PDF invoices automatically.

4

Add Services Catalog

Go to More → Service Parts. Add the services you offer (e.g., Oil Change, Wheel Alignment). You can also bulk-import via a CSV file.

5

Add Team Members

Go to More → Garage Users → Add User. Enter the mechanic's phone number and name. They can now log in with the Member role.

5.2 Creating a Repair Order

1

Go to Service Tab

Tap the Service tab from the bottom navigation bar.

2

Tap "New Order"

Press the + button or "New Order" from the Service Home screen.

3

Select or Add Customer

Search for an existing customer by name or phone, or add a new customer. You can import contact details directly from your phone's contacts.

4

Select or Add Vehicle

Choose the customer's vehicle from their saved vehicles, or add a new one (brand, model, registration number, variant).

5

Add Services & Parts

Select services from your catalog. Add parts used (with quantity and price). Enter any customer notes or instructions.

6

Assign a Mechanic (Optional)

Assign the order to one of your team members. They will receive it in their "My Work" tab.

7

Submit the Order

Tap "Create Order". The order is now live and appears in the Orders list.

5.3 Generating an Invoice

1

Open a Completed / Ready Order

Go to Service → Orders. Tap the order marked "Vehicle Ready" or "Completed".

2

Review Line Items

Verify all services and parts are listed correctly. Add labour charge %, tax %, or discount if applicable.

3

Generate Invoice

Tap "Generate Invoice". The invoice is created with a unique invoice number, your garage logo, customer details, vehicle info, and all charges.

4

Download or Share

Tap "Download PDF" to save, or "Share via SMS" to send a text summary directly to the customer's phone. The PDF includes your garage logo and all billing details.

5.4 Inventory Management

  • Parts Tab → Parts Home: View current stock levels for all parts
  • Inventory Alerts: See parts that are low in stock or out of stock
  • Purchase Orders: Create orders to vendors to restock parts
  • Stock In: Record received stock against purchase orders
  • Counter Sale: Sell parts directly over the counter (without repair order)

6 Member (Mechanic) Portal — User Guide

Who uses this: Garage technicians and mechanics. Log in with the phone number added by the garage owner.

6.1 Home Screen

The Member Home shows a personal dashboard with:

  • Pending: Number of orders assigned but not yet started
  • In Progress: Number of jobs currently being worked on
  • Completed (Total): All-time completed jobs count
  • Completed (Today): Jobs finished today
  • Total Earnings: Total billing amount from completed work

6.2 My Work Tab

The work screen has two tabs:

TabWhat It Shows
My AssignedOrders specifically assigned to this mechanic
All ActiveAll unassigned and in-progress orders in the garage

6.3 Accepting / Rejecting an Order

1

Find the Order

Go to Work → My Assigned tab. Orders assigned to you appear here with a yellow accent bar.

2

Accept or Reject

Tap Accept to take on the job, or Reject if you cannot take it. Rejected orders become available for other mechanics.

3

Start Work

Tap Start Work to change the status to "In Progress". The owner can see this update in real-time.

4

Mark Vehicle Ready

Once repair is complete, tap Mark Ready. The status changes to "Vehicle Ready" so the owner can notify the customer.

5

Complete

After the customer collects the vehicle, tap Complete to close the order.

6.4 Recording Parts Used

Inside an order detail screen, tap Edit Parts to add or update parts consumed in the repair. Enter part name, quantity, and price. This updates the billing automatically.

6.5 Inventory Tab

View the current stock of all parts available in the garage. Use this to check if a part is available before requesting it from the owner.

6.6 History Tab

View all previously completed orders. Useful for referencing past work, troubleshooting recurring issues, or tracking personal work history.

The Customer Home displays a personalised dashboard showing:

  • Active orders (in-progress or waiting)

Browse Available Services

Tap the Services tab to see all services offered by the garage with their prices.

2

Book a Service

Select the service you need. Choose your vehicle (or add a new one). Submit a repair request with any notes for the mechanic.

Open Invoices Tab

2

Download PDF

Tap Download PDF at the bottom of the invoice screen. A professionally formatted PDF is generated with the garage logo, all line items, and totals. Share or save it to your device.

4

Share via SMS

Tap Share SMS to open your SMS app with a pre-filled text summary of the invoice ready to forward to anyone.

7.5 Profile Tab

1

Edit Your Profile

Tap the edit icon (pen) in the top-right of the profile hero. Edit your full name, address, and state. Tap Save Changes.

2

Add a Vehicle

Scroll to "My Vehicles" section. Tap + Add. Enter the vehicle brand, model, registration number, and variant. Tap "Add Vehicle".

3

Logout

Scroll to the bottom of the Profile screen. Tap Logout to securely end your session.

11 Security

🔑

OTP Authentication

Phone-based OTP. No passwords stored. OTP valid for 5 minutes, max 5 attempts before lockout.

🎫

JWT Tokens

Short-lived access token + long-lived refresh token. Auto-refreshed on expiry. Revoked on logout.

🛡️

Role-Based Access Control

Every protected route checks user role. Owners cannot access member endpoints and vice versa.

⚙️

Helmet Security Headers

HTTP security headers set automatically: XSS protection, content-type sniffing, frame options.

🚦

Rate Limiting

Maximum 500 requests per 15 minutes per IP address to prevent abuse.

✅

Input Validation (Zod)

All API inputs validated before reaching controllers. Malformed data is rejected with specific error messages.

12 Setup & Deployment

12.1 Backend Setup

1

Prerequisites

Node.js 18+, MongoDB 6+ (local or MongoDB Atlas), npm

2

Install Dependencies

cd garage-system-backend && npm install

3

Configure Environment Variables

Create a .env file with:
MONGO_URI=mongodb://localhost:27017/apnogarage
JWT_SECRET=your-secret-key
JWT_REFRESH_SECRET=your-refresh-secret
PORT=5000
OTP_SERVICE_KEY=your-sms-api-key

4

Start Server

npm run dev (development) or npm start (production)

12.2 Mobile App Setup

1

Prerequisites

Node.js 18+, Expo CLI (npm install -g expo-cli), iOS Simulator or Android Emulator or Expo Go app on a physical device

2

Install Dependencies

cd garage-system-app_23_3 && npm install

3

Set Backend URL

Update the API base URL in src/api/axiosClient.js to point to your deployed or local backend server.

4

Start the App

npx expo start → Scan the QR code with Expo Go on your device, or press i for iOS / a for Android emulator.

5

Build for Production

Use EAS Build: eas build --platform android or eas build --platform ios
